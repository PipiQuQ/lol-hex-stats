# LOL海克斯乱斗战绩统计系统设计文档

## 项目概述

为LOL国服海克斯乱斗自定义5v5对局设计一个自动统计战绩、计算玩家评级的H5网页系统。

### 目标用户
- 20人左右的玩家群体，人员不固定
- 通过微信群分享链接查看战绩

### 核心功能
- 自动抓取玩家战绩数据
- 多维度玩家评级展示
- 每日最佳玩家评选
- 数据趋势图表

## 综合评分算法

### 设计思路
不同位置/英雄有不同的评估标准，系统根据英雄定位自动调整权重。

### 英雄定位分类
- **坦克**：承伤、控制时长、助攻权重更高
- **输出**：伤害、击杀权重更高
- **辅助**：助攻、治疗、控制权重更高
- **刺客**：击杀、伤害权重更高

### 评分公式

```
综合评分 = 胜率分(40%) + 表现分(40%) + 稳定性分(20%)

胜率分 = 胜率 × 100（如68%胜率 → 68分）

表现分 = 根据英雄定位加权计算
  - 输出型: 伤害30% + KDA40% + 助攻20% + 经济10%
  - 坦克型: 承伤35% + KDA30% + 助攻25% + 控制10%
  - 辅助型: 助攻35% + 治疗/护盾30% + KDA20% + 控制15%

稳定性分 = 最近10场胜率波动越小分越高
  - 波动小于10%: 满分
  - 波动10-20%: 80分
  - 波动20-30%: 60分
  - 波动大于30%: 40分
```

### 场次门槛
不足**5场**的玩家，评分显示灰色，提示"样本不足"

## 每日最佳算法

当日有对局的玩家中评选：
- 计算当日场均表现分
- 场次≥2场才有资格评选
- 综合考虑：当日胜率 + 当日表现分
- 展示：玩家名、当日评分、当日战绩

```
每日最佳评分 = 当日胜率分(50%) + 当日表现分(50%)
```

## 数据抓取方案

### 自动抓取
- 调研腾讯掌盟API或第三方数据源
- 通过玩家ID批量拉取历史对局数据
- 过滤条件：自定义房间 + 海克斯乱斗模式

### 降级方案
若API不可用，提供管理后台：
- 添加玩家：输入游戏ID，系统查询并确认身份
- 手动录入：批量粘贴对局数据或Excel导入
- 数据校验：检测重复录入、异常数据提示

### 数据更新
- 定时更新：使用Vercel Cron Jobs，每日凌晨3点自动拉取
- 手动触发：前端按钮触发即时更新
- 数据缓存：计算结果缓存到数据库
